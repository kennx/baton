# 红外设备扫描与控制系统设计文档

**日期**: 2026-05-19  
**目标平台**: M5StickS3 (ESP32-S3, ST7789 135×240 屏幕)  
**框架**: Arduino + M5Unified + M5GFX  

---

## 1. 设计目标

开发一个运行在 M5StickS3 上的通用红外设备扫描与控制程序：

- **接收学习**：持续监听环境中的红外遥控信号，自动解码、去重、保存
- **主动控制**：浏览已保存的信号列表，选择后通过红外 LED 重放发射
- **信号管理**：对保存的信号进行重命名、归类、删除
- **统一交互**：单按 btnA 移动选择，长按 btnA（>500ms）确认/进入

---

## 3. 菜单结构（状态机）

四个顶层状态对应主菜单，单按 btnA 在菜单项间循环移动，长按确认进入：

```
主菜单
├── 🔍 扫描模式
├── 📡 控制模式
├── 📁 信号管理
└── ⚙️ 设置
```

**全局 btnA 行为约定**：

| 场景 | 单按 btnA | 长按 btnA |
|------|-----------|-----------|
| 主菜单 | 向下移动选择项（循环） | 进入选中的子模式 |
| 子模式列表中 | 向下移动选择项 | 执行当前项的确认操作 |
| 子模式最底部项 | 移动到第一项（循环） | 回到主菜单 |
| 弹窗/确认框中 | 在选项间切换 | 确认当前选项 |

## 4. 扫描模式

### 4.1 界面

```
┌─────────────────────────┐
│ 🔋85%  信号:12  🔴扫描中 │
├─────────────────────────┤
│                         │
│    等待红外信号...       │
│                         │
│    ─────────────────    │
│    上次: NEC 0x20DF     │
│    地址: 0x10  命令:0x0B │
│    已存为: Signal-12    │
│    ─────────────────    │
│                         │
│    自动保存 ✓           │
│                         │
├─────────────────────────┤
│  短按:选择  长按:确认    │
└─────────────────────────┘
```

### 4.2 扫描状态机

| 状态 | 说明 | 屏幕显示 |
|------|------|----------|
| `IDLE` | 正常监听中 | "🔴 扫描中..." |
| `RECEIVED` | 刚收到并解码成功 | 显示协议+地址+命令，底部"已自动保存" |
| `RECEIVED_NEW` | 收到全新信号（与已有不重复）| 同上，但显示"新信号！" |
| `RECEIVED_DUP` | 收到与已有信号完全相同 | 显示"重复信号 — 跳过" |
| `FULL` | 存储已满（50个）| "存储已满！请删除旧信号" |

### 4.3 自动保存逻辑

1. 红外接收器持续监听（IRremoteESP8266 `IRrecv`）
2. 收到信号后解码，提取：协议类型、地址、命令、原始时长数组
3. **去重检查**：与已保存信号对比（协议+地址+命令），相同则跳过
4. 新信号自动生成名称 `Signal-{N}`（N = 当前总数 + 1），分类为"未分类"
5. 写入 LittleFS 存储，同时更新内存中的信号列表
6. 显示收到信号信息 2 秒后回到 `IDLE`
7. 收到信号时：屏幕边框绿色闪烁 + 蜂鸣器短鸣 50ms（蜂鸣器开启时）

### 4.4 快捷跳转

- 扫描模式下**单按 btnA** → 切换到控制模式（不用先回主菜单）
- **长按 btnA** → 回到主菜单

### 5.2 分类筛选

单按移动到列表最底部后，再按一次进入分类筛选：

```
┌─────────────────────────┐
│ 🔋85%  📡控制  筛选    │
├─────────────────────────┤
│  > 全部 (12)           │

└─────────────────────────┘
```

长按确认分类后，列表只显示该分类下的信号。

### 5.3 发射流程

1. 在列表中选中某信号，**长按 btnA** 触发发射
2. 屏幕显示发射中状态（信号名称、协议信息、进度条）
3. 发射完成后显示结果 0.5 秒：
   - "✓ 发送成功"
   - "✗ 发送失败"（如数据损坏）
4. 自动回到列表，焦点保持在刚才发射的信号上
5. 根据设置中的"发射重复次数"连续发射多次

### 5.4 快捷操作

- **长按 btnA** 在列表第一项时 → 回到主菜单

### 6.2 重命名（预设名称库）

M5StickS3 无键盘，采用**预设名称库**方式，比逐字符输入更实用：

**步骤 1 — 选择分类**：

```
┌─────────────────────────┐
│ 🔋85%  重命名 Signal-01 │
├─────────────────────────┤
│  > 电视                │
│    空调                │
│    风扇                │
│    机顶盒              │
│    自定义短名           │
├─────────────────────────┤
│ 短按:下选 长按:确认    │
└─────────────────────────┘
```

**步骤 2 — 选择名称**：

| 分类 | 预设名称 |
|------|----------|
| 电视 | 电源、音量+、音量-、频道+、频道-、静音、菜单、上、下、左、右、确认、返回 |
| 空调 | 电源、制冷、制热、除湿、自动、升温、降温、风速、扫风、定时 |
| 风扇 | 电源、风速、摇头、定时 |
| 机顶盒 | 电源、菜单、回看、直播、上、下、左、右、确认、返回 |
| 自定义 | 自定义-01 ~ 自定义-10 |

选择后自动保存，格式为 `{分类}-{名称}`，如 `电视-电源`、`空调-制冷`。

### 6.3 归类到文件夹

操作方式与重命名选分类类似，单按选择分类，长按确认移动。目标分类：电视 / 空调 / 风扇 / 机顶盒 / 未分类。

## 7. 数据存储

### 7.1 存储方案

使用 **LittleFS**，原因：
- 比 SPIFFS 更可靠，掉电不易损坏
- 适合小文件频繁读写
- PlatformIO 原生支持

### 7.2 文件结构

```
/littlefs/
  signals.json       ← 信号索引（名称、分类、协议、地址、命令、数据文件路径）
  signal_000.raw     ← 信号0的原始时长数组（二进制，uint16_t 微秒级）
  signal_001.raw
  ...
```

### 7.3 索引格式

```json
{
  "version": 1,
  "count": 3,
  "signals": [
    {
      "id": 0,
      "name": "Signal-01",
      "category": "未分类",
      "protocol": "NEC",
      "address": "0x20DF",
      "command": "0x10",
      "data_file": "signal_000.raw",
      "length": 67,
      "created_at": "2024-01-15T10:30:00"
    }
  ]
}
```

### 7.4 容量限制

- 最大信号数：**50 个**
- 每个信号原始数据上限：约 300 个时长（足够覆盖空调长码）
- 总存储占用：约 40KB（远小于 8MB Flash）
- 删除信号时同步释放对应的 `.raw` 文件

## 9. 红外协议支持

使用 **IRremoteESP8266** 库（ESP32 兼容），支持接收和发射：

- **解码协议**：NEC、Sony、Samsung、LG、RC5、RC6、Panasonic、Sharp、JVC、Denon、Whynter 等 40+ 种
- **未知协议**：若无法解码，保存原始时长数组，支持原始重放
- **发射方式**：通过库内置的 `IRsend` 按协议精确发射，未知协议用 `sendRaw()` 重放

## 11. 风险与注意事项

1. **红外接收引脚**：M5StickS3 的红外接收模块引脚需在代码中正确配置，待实现时确认硬件 PinMap
2. **发射距离**：红外 LED 的发射距离受限于硬件电路，程序无法控制功率，发射重复次数可补偿部分设备的响应灵敏度
3. **屏幕刷新率**：扫描模式下频繁的信号接收和 UI 更新需合理限制刷新频率，避免性能问题
4. **LittleFS 格式化**：首次运行若未格式化，程序需自动初始化文件系统
